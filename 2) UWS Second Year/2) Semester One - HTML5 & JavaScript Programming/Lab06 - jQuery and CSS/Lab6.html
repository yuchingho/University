CSS Style-sheets

A guide to Cascading Style Sheets

Cascading Style Sheets (CSS) is a language used to assign formatting to HTML documents. Styles are defined as simple (textual) rules that indicate how particular elements are to appear in the HTML mark-up. There are three ways in which style information can be added to an HTML document:

  • Inline styles, in which styling information is added directly to HTML elements in the mark-up. You have already done this using the 'style' attribute.
  • Embedded styles, which are blocks of CSS information added to the HTML document (usually in the <head> section
  • External style-sheets, which are seperate files (usually with .css appended to the end of their name) which defines families of style rules that can be applied to HTML documents in a more structured way.

Style Priorities

Style-sheets rules are applied to elements in a document, and will then automatically apply to any child elements. For example, a style rule applied to a <table> element will automatically apply to all of the child elements of the table - its heading row, the table rows and the cells within a row. However, a more specific rule can then be applied to child elements, so a <tr> rule will override the corresponding parent table's rules.

Styling Forms

Form elements can also be styled using CSS rules. This can be used to highlight specific elements, align form elements more clearly or even dynamically change elements (using some Javascript code to apply different styles to elements in particular states.
